Instruction

Submit your solution files by sending emails to rem@hust.edu.cn.
The subject of the email should include the keyword [NPBenchmark].
Each attachment of the email should be a solution file generated by the SDKs.
Find the SDKs prefixed with NPBenchmark at github.com/HUST-Smart.

Specification

The SDK should append a single-line header to the solution file.
The SDK should set the 'problem', 'instance' and 'duration' fields in the solution header and guide users to fill other fields.
The checker should accept two command line arguments which are the paths of the instance and solution files, respectively.
The checker should print the obj value (a positive number) to the standard output if the solution is valid, otherwise anything except a positive number should be printed.
